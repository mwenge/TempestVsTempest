\chapter{wells}
\label{sec:wells}
\lhead[tempest]{}

Wells are fun and should be easy to draw. To start with, we can describe a 2d shape
of our choice using a bunch of x/y co-ordinates as points.
\begin{figure}[H]
    \centering
    \begin{adjustbox}{width=7cm,center}
      \includegraphics[width=12cm]{src/wells/HEART_dots_no_title.png}%
    \end{adjustbox}
  \caption{Make our dots.}
\end{figure}

Next we can join these together to give us a 2d shape.
\begin{figure}[H]
    \centering
    \begin{adjustbox}{width=8cm,center}
      \includegraphics[width=12cm]{src/wells/HEART_2d.png}%
    \end{adjustbox}
  \caption{Start with a 2d shape..}
\end{figure}

To make this thing three dimensional all we have to do is project our shape onto
some chosen distant position along the Z plane.
\vspace{-0.5cm}
\begin{figure}[H]
    \centering
    \begin{adjustbox}{width=8cm,center}
      \includegraphics[width=12cm]{src/wells/HEART_3d.png}%
    \end{adjustbox}
  \caption{..then make it three dimensional.}
\end{figure}

Seems simple enough. With this as our objective the data structure below gives us
everything we need to create the web. In addition to listing the 16 vertices that
make up the web's shape in two dimensions.

\begin{lstlisting}
; Data structure for the 'HEART' well. 
; X Co-Ordinates
    .BYTE 0DA,0A4, 87, 80, 79, 5C, 26, 10    ; HEART
    .BYTE  10, 20, 48, 80,0B8,0E0,0F0,0F0
; Y Co-Ordinates
    .BYTE 0E4,0E8,0B7, 80,0B7,0E8,0E4,0B2    ; HEART
    .BYTE  7A, 47, 20, 10, 20, 47, 7A,0B2
\end{lstlisting}

To help you relate this table to the co-ordinates in our graphs let's list
it in decimal instead:
\begin{lstlisting}
; Data structure for the 'HEART' well. 
; X Co-Ordinates
    .BYTE 218,164,135,128,121, 92, 38, 16    ; HEART
    .BYTE  16, 32, 72,128,184,224,240,240
; Y Co-Ordinates
    .BYTE 228,232,183,128,183,232,228,178    ; HEART
    .BYTE 122, 71, 32, 16, 32, 71,122,178
\end{lstlisting}

To understand how we turn this list of numbers into a well, we're goint to look
at the routine that is used to draw a miniature version on the level selection screen.
The procedure consists of reading each X and Y co-ordinate pair in the order that they're
given, starting with \icode{218} for X and \icode{228} for Y, and drawing a line to the 
position given by the next pair. As we saw in our 
\hyperref[sec:cursors]{\textcolor{blue}{'cursors'}}
chapter, we do this by constructing 'vector' commands and adding them to the vector list.
Each command will consist of the X and Y direction in which to move the beam. In order to
construct this list of vector commands we must loop through our co-ordinates above. In the
routine given below, these co-ordinates are stored in \icode{NEWLIX} (for the X co-ordinates)
and \icode{NEWLIZ} (for the Y co-ordinates).

\begin{lstlisting}
        .SBTTL  UTILITY-DRAW WELL SHAPE
DSPHOL:
        ..
        BEGIN            ; LOOP FOR EACH PT ON EDGE
        LDY SAVEY        ; GET THE CURRENT INDEX
        LDA Y,NEWLIX     ; STORE CURRENT X CO-ORD IN A
        TAX              ; MOVE IT TO X
        SEC              ; SET THE CARRY BIT.
        SBC PXL          ; SUBTRACT THE LAST X POSITION.
        PHA              ; STORE THE X MOVEMENT ON THE STACK.
        STX PXL          ; STORE THE CURRENT VALUE AS THE LAST X POSITION.
        LDA Y,NEWLIZ     ; STORE CURRENT Y CO-ORD IN A.
        TAY              ; MOVE IT TO Y
        SEC              ; SET THE CARRY BIT.
        SBC PYL          ; SUBTRACT THE LAST Y POSITION.
        TAX              ; STORE THE Y MOVEMENT IN X.
        STY PYL          ; STORE THE CURRENT VALUE AS THE LAST Y POSITION.
        PLA              ; PUT OUR X MOVEMENT IN A.
        ; WE NOW HAVE THE X MOVEMENT IN A AND THE Y MOVEMENT IN X.
        ; WE CAN DRAW OUR VECTOR LINE.
        JSR VGVTR1       ; DRAW OUR VECTOR.
        DEC SAVEY        ; DECREMENT THE INDEX TO MOVE TO NEXT CO-ORDINATE.
        MIEND            ; LOOP UNTIL WE'RE DONE.
\end{lstlisting}

The procedure consists of calculating an X and Y movement for each vector line by 
subtracting the last X and Y position in the co-ordinate list from the current one.
This difference or 'delta' tells the vector ROM the number of units in the X and Y
direction it should move the line. The end result is one single face of our web
in two dimensions.

as follows. In each case we list the actual vector instruction used, encoded into two bytes
