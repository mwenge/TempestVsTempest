of our choice using a bunch of x/y co-ordinates as points.

some chosen distant position along the Z plane.

Seems simple enough. With this as our objective the data structure below gives us
everything we need to create the web. In addition to listing the 16 vertices that
make up the web's shape in two dimensions, we also include some information on whether
the web should be open (like the one pictured above) or closed (like a circle or 
square). Since a web of 16 vertices will give us 14 lanes which the player and
enemies can occupy we also add a 'Rotation Table' to tell us how players and
enemies should be oriented when on that lane.

\begin{lstlisting}
; Data structure for the 'Sine Wave' web. 
web13:       
  dc.w 14    ; Number of lanes in the web.
  dc.w  6    ; The lane the player starts on.

  ; The x/y pairs of all vertices in the web. So for example
  ; -2/14 indicates the X and Y co-ordinates of the first vertex.
  ; There are always 16 in total.
  dc.w -2,14,-2,12,-1,10, 1, 8
  dc.w  3, 7, 5, 7, 7, 8, 9,10
  dc.w 11,12,13,13,15,13,17,12
  dc.w 19,10,20, 8,20, 6,-2,14

  dc.w 0     ; 0 = Open Web, -1 = Closed Web
    
  ; Rotation Table (angle of an object within a particular lane)
  ; The length of this list is specified by the '14' above.
  dc.w -64,-48,-32,-16, 0,16,32,32,16, 0,-16,-32,-48,-64
\end{lstlisting}

The values given in the Rotation Table are angles of roll to apply to the claw
(and to the claw's enemies) when placing them in the lane. Negative values rotate
the object to the right, while positive values rotate it to the left.

This is the important part of the routine that converts the web data structure into a list of vertices and lines between vertices
that will make up the web itself:

\begin{lstlisting}
  ; Read in the x/y pairs from our data structure. We create an interleaved
  ; list of near and far points in a2 and a 'connlist' in a3 that describes
  ; how each point is connected to the others.
xweb:
  ; Get the current x and y pair
  move (a1)+,d0    ; Store the Y value in d0.
  move (a1)+,d1    ; Store the X value in d1.
  ext.l d0         ; Make d0 4 bytes long.
  ext.l d1         ; Make d1 4 bytes long.

  ; Check if this is the largest X value so far
  cmp d5,d1       ; Compare X with the largest so far, stored in d5. 
  blt xweb2       ; If it's less, skip to xweb2 below.
  move d1,d5      ; It's bigger, so save it in d5.

xweb2:
  ; Store the x,y,z value for the near point in the web
  move.l d0,(a2)+ ; Y value
  move.l d1,(a2)+ ; X value
  clr.l (a2)+     ; Z value for near point (always 0)

  ; Store the x,y,z value for the far point in the web
  move.l d0,(a2)+ ; Y value
  move.l d1,(a2)+ ; X value
  move.l d7,(a2)+ ; z value for far point (calculated by initvo).

  ; a3 is the 'connlist', the list describing how each point is connected
  ; to the others. 

  ; Create entries in the connlist for this point. First we create an entry
  ; for the 'near' point.  This will be connected to the far point and the
  ; next point in the web data structure. Each entry starts with an index
  ; to the point in question, followed by indices to the points its connected to.
  move d3,(a3)+   ; Make this a connlist entry for the 'near' point. 
  move d3,d4      ; Store the 'near' point index in d4. 
  addq #1,d4      ; Add 1 to it, so it's now an index to the 'far' point.
  move d4,(a3)+   ; Store this as first connected point in the entry.
  addq #1,d4      ; Add 1 to d4 again so it's an index to the next point.
  move d4,(a3)+   ; Store this as the second connected point in the entry.
  move #0,(a3)+   ; Mark the end of the entry with a zero.

  ; Create an entry in the connlist for the 'far' point.
  subq #1,d4      ; Subtract 1 from d4 so it's an index to the far point.
  move d4,(a3)+   ; Make this an entry for the 'far' point.
  addq #2,d4      ; Add 2 to d4 so it's an index for the next 'far' point.
  move d4,(a3)+   ; Store this as the connected point in the entry.
  move #0,(a3)+   ; Mark the end of the entry with a zero.
  move.l a3,(a5)+ ; Add our entries to the overall connlist list of lists.
  add #2,d3       ; Point d3 to the next X/Y pair.

  ; Keep looping until we finish reading the X/Y pairs in the data structure.
  dbra d6,xweb    ; Get the next pair
\end{lstlisting}

Its job is to read in X and Y pairs in our \icode{web13} data structure. To make sense of them
as describing a web it uses the pairs to create a connection list or \icode{connlist} that describes
how each of the points in the web are connected to each other. Let's take a look at how the loop in
\icode{xweb} builds up this \icode{connlist} as it iterates over the X and Y pairs in \icode{web13}.

In this first iteration through \icode{xweb} we've managed to construct the first three
lines of our web. On the near and far side we've connected the first two points in the web,
and we've connected the near and far side of the first point. 

In our \icode{connlist} the entry
\icode{(1,3,0)} connects the far point (\icode{1}) with the next far point (\icode{3}). The \icode{0}
at the end is a terminator sentinel telling us it's the end of that particular entry.

The first entry in the \icode{connlist} tells us that the first near point (\icode{0}) is connected to
its far point counterpoint (\icode{1}) and the the next near point (\icode{2}). Again the zero at the
end is a terminator for the entry.

Moving to the next X/Y pair in our next diagram we repeat this process, layering on the same shape and
the same set of connections between the near and far points for the next point and its successor.
